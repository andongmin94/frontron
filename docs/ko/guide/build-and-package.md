# 빌드와 패키징

개발 모드가 잘 동작한다면 다음 단계는 배포 가능한 결과물을 만드는 것입니다.

이 페이지의 목표는 `npm run build` 가 무엇을 하는지와 결과물이 어디에 생기는지를 설명하는 것입니다.

## 1. 명령

```bash
npm run build
```

스타터에서는 이 명령이 아래로 연결됩니다.

```bash
npm run app:build
```

결국 `frontron build` 가 실행됩니다.

## 2. 빌드 중에 무엇이 일어나나요?

빌드 흐름은 아래와 같습니다.

1. 렌더러 빌드
2. `.frontron/` 아래에 runtime 파일 stage
3. 데스크톱 앱 패키징

런타임과 패키징 로직은 복사된 템플릿 파일이 아니라 `frontron` 이 소유합니다.

## 3. 빌드 전에 확인하면 좋은 것

- `npm run app:dev` 가 한 번은 정상적으로 동작했는지
- 아이콘이나 앱 메타데이터를 바꿨다면 저장했는지
- 터미널에 남아 있는 runtime 에러가 없는지

## 4. Windows 에서 어떤 결과를 기대할 수 있나요?

현재 기본 설정에서는 주요 결과물이 `output/` 아래에 생성됩니다.

보통 아래 항목을 보게 됩니다.

- `win-unpacked/`
- 설치용 `.exe`

정확한 파일 이름은 앱 이름과 버전에 따라 달라질 수 있습니다.

## 5. 빌드 후 어디를 보면 되나요?

먼저 아래 폴더를 보세요.

```text
dist/
.frontron/
output/
```

- `dist/`: 빌드된 웹 프론트엔드
- `.frontron/`: Frontron staging 과 생성 파일
- `output/`: 패키징된 데스크톱 결과물

::: tip
빌드는 성공했는데 폴더가 헷갈린다면, 다음 페이지에서 각 폴더를 하나씩 보면 이해가 쉬워집니다.
:::

::: warning
Windows 에서는 프로젝트 경로가 너무 깊으면 패키징이 깨질 수 있습니다. 긴 패키징 경로 안에서 파일을 찾지 못하는 에러가 보이면 `C:\dev\my-app` 같은 더 짧은 경로로 옮겨서 다시 빌드해 보세요.
:::
