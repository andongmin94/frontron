# 생성된 프로젝트 구조 이해하기

처음에는 모든 파일을 다 읽을 필요가 없습니다.

Frontron이 만든 프로젝트를 처음 볼 때는 "어떤 폴더가 무슨 역할을 하는지"만 이해해도 충분합니다.

## 1. 가장 먼저 볼 폴더

```text
src/
  electron/
  components/
public/
package.json
```

## 2. `src/electron/`

이 폴더는 Electron 쪽 코드를 모아 둔 곳입니다.

초보자는 아래 파일 이름부터 익히면 됩니다.

- `main.ts`: 앱 시작점
- `preload.ts`: 렌더러에 안전하게 기능을 노출하는 브리지
- `ipc.ts`: 창 제어 같은 통신 처리
- `window.ts`: 메인 창 생성
- `tray.ts`: 시스템 트레이

## 3. `src/components/`

이 폴더는 화면에 보이는 React 컴포넌트가 들어 있는 곳입니다.

예를 들어:

- `TitleBar.tsx`: 창 상단 커스텀 타이틀바
- `ui/`: 미리 들어 있는 UI 컴포넌트 모음

처음에는 모든 `ui/` 컴포넌트를 읽을 필요가 없습니다. 지금은 "이미 준비된 UI 조각이 많다" 정도만 알면 충분합니다.

## 4. `public/`

정적 파일이 들어 있는 폴더입니다.

초보자에게 가장 중요한 파일은 `icon.ico`입니다.

## 5. `package.json`

이 파일은 아래 두 관점에서 중요합니다.

- 어떤 명령으로 앱을 실행/빌드하는지
- 패키징에 어떤 이름과 아이콘을 쓸지

즉, 처음 쓰는 사람에게는 "실행 명령과 빌드 설정을 보는 곳"이라고 이해하는 게 가장 쉽습니다.

## 6. 지금은 무엇을 몰라도 괜찮나요?

처음 단계에서는 아래를 몰라도 괜찮습니다.

- 모든 Electron 라이프사이클 이벤트
- UI 컴포넌트 내부 구현
- 빌드 도구의 세부 옵션

지금 필요한 것은 "어떤 문제를 만나면 어떤 폴더를 먼저 보면 되는지"를 아는 감각입니다.
